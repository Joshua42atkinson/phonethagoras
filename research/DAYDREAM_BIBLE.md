# The Daydream Bible
*Architectural & Pedagogical Governance for the Sovereign Homeschool Sandbox*

> **Identity:** The System Blueprint
> **Purpose:** To codify the quantitative engineering of meaning-making.

---

## 1. Market Reality & The Sovereign Pivot
The modern homeschool market is exhausted by cloud-tethered, telemetry-heavy "edtech" that acts as a surveillance layer between parent and child. 
Daydream is engineered for **absolute sovereignty**. 

*   **Zero-Cloud Constraint:** The entire system compiles to a single binary. 
*   **Parent as SME:** We do not supply the curriculum; we supply the *authoring engine*.
*   **The Problem with "Telling":** Most educational software *tells* the student what to think via linear videos and quizzes.
*   **The Daydream Solution ("Showing"):** We use a **Branching Somatic Graph**. The system presents an environment (Image + Sound + Story) and forces a *quantifiable choice*.

## 2. Quantitative Meaning-Making (Show, Don't Tell)
How do we measure "meaning"? We cannot measure the soul, but we can measure the *somatic gateway* and the *topological traversal* of ideas.

### The Somatic Gate (The PLING!)
Before a child can make a choice, they must be present.
*   **The Metric:** Pitch-matching via the Web Audio API. 
*   **The Execution:** A slide plays a 432Hz backing frequency. The choices remain locked until the child vocalizes the frequency. 
*   **The Meaning:** This forces a pause. It breaks the "scroll-and-click" dopamine loop and grounds the nervous system. You cannot mindlessly click through Daydream.

### Topological Choice (The Stories We Tell Ourselves)
Every node in the graph represents a "Thought" or an "Emotion." 
The branches represent "The Stories We Tell Ourselves."
*   If the slide subject is **"Frustration,"** the choices are not "Right" or "Wrong."
*   Choice A leads to the **"Resilience"** node.
*   Choice B leads to the **"Withdrawal"** node.
By graphing the child's path through the sandbox, the parent can visually *show* the child the narrative they are building, organically introducing the concept of cognitive bias.

## 3. The Architecture of a Slide
A Daydream slide is the absolute minimum viable container for cognitive load. It consists of exactly four elements:

1.  **The Subject Word (VAAM):** The active anchor concept (e.g., *Bias*, *Ecosystem*, *Patience*).
2.  **The Ambient Image:** A single, static visual setting the environmental context.
3.  **The Backing Audio:** A drone, tone, or simple song setting the somatic frequency.
4.  **The Story & Choices:** The narrative prompt and the branching paths forward.

There are no flashing animations. There are no scoreboards. There is only the environment, the breath (vocalization), and the choice.

## 4. Document Discipline
This project maintains strict document discipline. 
*   **No Ego:** We do not write about how "smart" the code is. We write about what the code *does*.
*   **No Stubs:** If a feature is not fully implemented and rigorously tested, it is removed from the documentation. 
*   **Lifelong Learning:** The code itself must be readable, modular, and extensively commented, so that the act of modifying the sandbox becomes a shared learning experience for the parent and the child.
